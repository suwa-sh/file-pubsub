## 差分更新の的

REQ-018/SPEC-018-01 により、単一インスタンス前提(HAなし)を VIP + NFS 共有 data_dir 上の active/standby 自動フェイルオーバーへ見直す。これに伴い可用性カテゴリ(A)の冗長化・切替時間・回復目標、および C.3.3.1 障害復旧方式を再評価した。規模感(B 性能・拡張性等)は不変のため再推論しない。

## 前提（RDRA 差分 20260620_171535_add_redundant_failover より）

- 唯一性保証は方式B(lease 自動奪取・TTL 失効で standby 自動昇格)+ 方式A(外部クラスタ fencing 委譲。Pacemaker/keepalived、VIP + serve を同一リソースグループ)の併用。設定で切替・併用可。
- 前提: VIP + NFS(v4 推奨)共有 data_dir、NTP 時刻同期、single-writer(serve は常に1つ)維持。
- lease TTL は NFS 属性キャッシュ最大(actimeo 既定60s)より十分大きく設定でき既定も大きく取る → 切替時間は分オーダー。
- split-brain の窓でも AtomicWrite + at-least-once 冪等再開 + fail-closed 冪等照合により被害は高々1メッセージの重複配信に限定(破損・喪失なし)。exactly-once は保証せず既存 at-least-once 方針を継続(REQ-016)。
